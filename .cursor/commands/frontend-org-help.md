# Frontend Organization: Help

Display help and available commands for the Frontend Decision Intelligence Engineering Organization.

## Instructions

Present the organization overview and available commands.

```
═══════════════════════════════════════════════════════════════
   FRONTEND DECISION INTELLIGENCE ENGINEERING ORGANIZATION
                           HELP
═══════════════════════════════════════════════════════════════

This organization manages frontend development for the Water
Risk Platform decision interface.

AVAILABLE COMMANDS
──────────────────

@frontend-org-start-session
  Start a new working session with full startup protocol.
  Use for: New features, gap fixes, improvements.

@frontend-org-resume-session
  Resume an in-progress session.
  Use for: Continuing incomplete work.

@frontend-org-review-state
  Review organization state (gaps, debt, sessions).
  Use for: Understanding current status, planning work.

@frontend-org-adversary-review
  Run adversarial review of current frontend.
  Use for: Finding weaknesses, challenging assumptions.

@frontend-org-quick-gap
  Quickly start a session on a known gap.
  Use for: Addressing documented gaps efficiently.

@frontend-org-startup-check
  Validate organization setup without starting work.
  Use for: Verifying everything is configured correctly.

@frontend-org-propose-surface
  Propose a new decision surface.
  Use for: Adding new UI surfaces to the backlog.

IN-SESSION COMMANDS
───────────────────

During active sessions, you can use:

/question <text>  - Pause and ask a clarifying question
/status           - Show current session state  
/debate           - Surface internal specialist disagreement
/adversary        - Invoke devil's advocate review
/abort            - Terminate session with logged rationale

ORGANIZATION STRUCTURE
──────────────────────

frontdev/
├── governance/     - Protocols and standards
├── memory/         - Institutional knowledge
├── pods/           - Specialized teams
├── councils/       - Governance bodies
├── sessions/       - Working session artifacts
└── specialists/    - Role definitions

SPECIALIST ROLES
────────────────

• Lead         - Orchestrates sessions
• Engineer     - Implements code
• UI Designer  - Visual design
• UX Designer  - User experience
• Hierarchy    - Information architecture
• Performance  - Speed and efficiency
• Accessibility- WCAG and crisis usability
• Explainability- Transparency and audit
• Adversary    - Challenges assumptions

KEY PRINCIPLES
──────────────

1. Decision Primacy - Enable defensible, timely decisions
2. Truth Over Comfort - Never fabricate certainty
3. Urgency Is Visual - Time pressure must be felt
4. No Dashboard Theater - Compel action, don't just display

For full principles, see: frontdev/PRINCIPLES.md

═══════════════════════════════════════════════════════════════

Ready to start? Try: @frontend-org-start-session
═══════════════════════════════════════════════════════════════
```
